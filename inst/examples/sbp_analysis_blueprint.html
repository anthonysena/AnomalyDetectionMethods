Proposed Dataset Schema

Recommended row definition:

  • one row per SBP value x age band x gender x site
  • value is the SBP value you want to evaluate
  • frequency is the number of measurements with that exact SBP in that context
  • ageBand is the interpretable 5-year bucket
  • ageMid is the numeric midpoint used as a feature when you want age to remain in the model
  • gender and site are used for stratification rather than numeric distance calculations

In this framing, the anomaly detectors should generally use:

  • featureColumns = c("value", "ageMid")

within a chosen (site, gender) stratum.

Preprocessing Steps

  1. Restrict to the intended blood-pressure measurement concept and unit.
  2. Remove implausible or clearly miscoded values if you want to focus on distributional anomalies rather than impossible values.
  3. Bin age into 5-year bands for interpretability and create ageMid for modeling.
  4. Standardize site and gender values.
  5. Aggregate to the row definition above.
  6. Run stratum diagnostics before fitting anomaly detectors.

How to read this:

  • distinctValues tells you whether the stratum has enough SBP support points.
  • totalFrequency tells you whether the stratum has enough mass to be worth modeling.
  • top1Mass and top5Mass tell you whether the distribution is overly concentrated.
  • supportsK_* tells you whether candidate LOF neighborhood sizes are even plausible.
  • usableForLof, usableForIf, and usableForGad are first-pass readiness flags, not guarantees.

Recommended Workflow For Real SBP Data

  1. Build the aggregated SBP dataset at value x age band x gender x site.
  2. Run summarizeStrataDiagnostics() across (site, gender) strata.
  3. Use flagAnalyzableStrata() to identify strata that are large and varied enough to support modeling.
  4. Within usable strata, run LOF, IF, and GAD on c("value", "ageMid").
  5. Use the existing tuning helpers to assess stability of:
    • LOF across k
    • IF across ntrees, sampleSize, and seeds
    • GAD across preprocessing and covariance choices
  6. Prioritize rows flagged consistently across methods or consistently across nearby settings.

Practical Caution

If you stratify by site, gender, and ageBand all at once, then age is no longer a varying feature within the stratum. In that case, the models are effectively working on SBP values alone. If you want age to remain a feature, stratify by site and gender, and let ageMid vary within that stratum.
