Purpose

This example shows how to learn a plausible value range from a single value-frequency distribution by:

  1. running several univariate outlier detectors
  2. converting their outputs into binary flags
  3. combining those flags by consensus voting
  4. deriving plausible bounds from the retained values

The demo distribution has:

  • a dense central region
  • thin but plausible tails
  • one obvious high-end implausible value

Useful columns:

  • consensusCount: number of methods flagging the value
  • consensusScore_count: plain vote count
  • consensusScore_frequency_weighted: vote count multiplied by frequency
  • consensusScore_support_weighted: vote count multiplied by relative support
  • isConsensusOutlier_*: whether the value is excluded under each consensus strategy

Reading The Differences

  • count consensus is the closest to a simple majority-voting approach.
  • frequency_weighted penalizes repeated flagged values more heavily.
  • support_weighted downweights rare values, which can help preserve unusual but plausible extremes.
  • weighted_quantile bounds are more robust than raw_min_max because they are less sensitive to single retained edge values.

Practical Interpretation

For measurement-range learning, the usual workflow is:

  1. start with consensusMethod = "count"
  2. use a moderate threshold such as 3
  3. compare weighted_quantile against raw_min_max
  4. inspect whether rare but clinically plausible values are being retained
  5. increase or decrease the consensus threshold depending on how conservative you want the learned range to be

Optional Next Step

If the distribution is strongly skewed, repeat the process after a log transformation and compare the resulting plausible bounds.
